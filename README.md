# B_parser — анализ длинных серий BTC для стратегии мартингейл на Polymarket

## Что это и зачем

Программа проверяет конкретную гипотезу: **можно ли по поведению BTC на
бирже отличить "нормальную" тройку одинаковых исходов (3 UP или 3 DOWN
подряд на 5-минутках) от начала длинной серии 7+, которая ломает мартингейл-стратегию на Polymarket**.

Идея стратегии:
- На Polymarket BTC 5-минутных маркетах ставим **после 3 одинаковых
  исходов подряд — против серии** (классический мартингейл).
- Делаем до 3 догонов (ставки 1, 2, 4), при проигрыше всей цепи теряем 7.
- При выигрыше — забираем чистую прибыль (выплата ×1.8 по умолчанию).
- Если серия идёт 7+ — мы в большом минусе.

Программа:
1. Скачивает исторические свечи (5-минутные) с трёх бирж (Binance Spot,
   Binance Futures, Bybit) — это бесплатно через публичные API.
2. Опционально — массив сделок (aggTrades с Binance) для расчёта
   taker imbalance.
3. Реконструирует UP/DOWN исходы (`close > open` в каждой 5-минутной
   свече — стандартное правило для Polymarket).
4. Находит **все серии** одинаковых исходов и **все точки входа** в
   стратегию (после 3-в-ряд).
5. Считает **признаки на момент входа**: пробои локальных hi/lo
   (30м/1ч/3ч/9ч/24ч/7д), объёмы 3 свечей vs средний за час, taker imbalance, форму свечей,
   расширение волатильности, расстояние от EMA20 и VWAP.
6. Прогоняет **бэктест с 10 фильтрами**:
   - 5 фильтров из ТЗ заказчика (пропуск при пробое/большом объёме/imbalance)
   - 5 "инвертированных" фильтров (пропуск при отсутствии этих условий —
     обратная гипотеза)
7. Делит данные на train/test по walk-forward (для проверки, что
   стратегия не переподогнана).
8. Выгружает всё в **Excel с описаниями на русском**.

### Что программа НЕ делает
- Не торгует автоматически.
- Не делает прогнозов на будущее.
- Не пишет в Polymarket напрямую (нет API-ключей, нет ордеров).
- Не использует платных данных или подписок.

---

## Что нужно перед запуском

- **Папка с `B_parser.exe` и `config.yaml` в одном месте** (например,
  `C:\BTC\`). Программа сама создаст рядом подпапки `cache/`, `export/`, `logs/`.
- **Доступ в интернет**. Если в РФ — нужен VPN, иначе биржи будут отдавать
  ошибку 403.
- ~500 МБ свободного места для кэша свечей. Если будешь качать сделки
  (aggTrades) — до 20 ГБ.

## Инструкция по использованию

1. **Положи `B_parser.exe` и `config.yaml` в одну папку**.
2. **Двойной клик на `B_parser.exe`** — откроется консольное окно.
3. Появится главное меню:
   ```
   Что делаем?
     1) Скачать данные (klines + aggTrades)
     2) Что лежит в кэше
     3) Прогнать анализ (UP/DOWN + серии + сигналы)
     4) Прогнать бэктест (стратегия + фильтры)
     5) Выгрузить всё в Excel
     6) Проверить связь с биржами
     7) Упаковать логи в ZIP для разработчика
     0) Назад
   ```
4. **Идёшь по пунктам сверху вниз**: 6 → 1 → 2 → 3 → 4 → 5.
   В любой момент можно вернуться в меню и повторить шаг.

### Шаг 6. Проверить связь с биржами

Жми `6` и Enter.

Ожидаемый результат:
```
  Binance Futures        HTTP 200  120 ms
  Binance Spot           HTTP 200  150 ms
  Bybit                  HTTP 200  130 ms
  data.binance.vision    HTTP 200  200 ms
```
Если у кого-то `HTTP 403` или `ОШИБКА` — **нужен VPN**, без него
дальнейшие шаги не сработают.

### Шаг 1. Скачать данные

Жми `1`. Программа спросит даты:

```
Старт (UTC, YYYY-MM-DD) [2025-12-18]:   ← Enter (дефолт OK)
Конец (UTC, YYYY-MM-DD) [2026-05-07]:   ← Enter (дефолт OK)
```

Затем выбор источников:

```
Какие источники?
  1) Всё (klines + aggTrades — долго)
  2) Только klines (быстро)                    ← начни с этого
  3) Только Binance Spot
  4) Только Binance Futures
  5) Только Bybit
  6) Только aggTrades (медленно, ГБ трафика)
```

В первый раз жми `2` — скачаются только свечи (5-15 минут).
Должно вывестись:
```
[binance_futures] cached rows: 40321
[binance_spot]    cached rows: 40321
[bybit_linear]    cached rows: 40321
```

### Шаг 2. Посмотреть, что в кэше

Жми `2`. Покажет:
```
[binance_futures] rows=40321  2025-12-18 -> 2026-05-07
[binance_spot]    rows=40321  ...
[bybit_linear]    rows=40321  ...
[binance_aggtrades_5m] empty
```

### Шаг 3. Прогнать анализ

Жми `3`. Программа спросит:

```
Источник для реконструкции исходов?
  1) binance_spot (рекомендую)            ← выбери 1
  2) binance_futures
  3) bybit_linear

Как считаем UP/DOWN?
  1) close > open в той же свече (стандарт)  ← выбери 1
  2) close[t] > close[t-1] (скользящее)

Сколько ставок догон после тройки? [3]:    ← Enter
С какой длины серия считается 'длинной'? [7]:   ← Enter
```

В консоли увидишь:
```

  Исходов     : 40321  UP=20189  DOWN=20132  (доля UP 50.1%)
  Серий       : всего=20605  макс_длина=15  ≥7: 272  ≥10: 25
  Сигналов    : всего=5053  побед=4514  проигрышей=539  win_rate=89.3%
  Доля длинных серий в сигналах (≥7): 272 (5.4%)

  Сравнение по признакам  (победы vs длинные серии):
    признак                                       победы   длинные   разница
    объём 3 свечей / 3x среднее за час            1.058     1.038    -0.020
    ...
```

### Шаг 4. Прогнать бэктест

Жми `4`. Параметры:

```
Какой источник сигналов?
  1) binance_spot (рекомендую)            ← 1

Выплата Polymarket (1.8 или несколько через запятую): ← 1.8
   ↑ Можно ввести список — например 1.8,1.85,1.9,2.0 — тогда программа
     прогонит несколько раз с разной выплатой и положит отдельный лист
     "Sweep_по_payout" в отчёт. Это удобно, чтобы понять, при каком
     payout стратегия становится прибыльной.

Базовая ставка [1]:                       ← Enter
Сколько ставок-догонов после тройки [3]:  ← Enter
Режим ставок?
  1) Удвоение (1, 2, 4...) — классический мартингейл  ← 1
  2) Фиксированная ставка
Граница train/test (UTC) [...]:            ← Enter
Папка для xlsx-отчёта [export]:            ← Enter
```

В консоли увидишь три таблицы — для всего периода, train и test.
Каждая показывает все стратегии: без фильтров, 5 из ТЗ, 5 «инвертированных»,
плюс фильтр «ночь МСК» и комбо. Для каждой — сколько сигналов осталось
после фильтра, win rate, PnL, max drawdown.

Параллельно создаются 7 диаграмм в `export\charts\` (см. раздел
«Графики» ниже).

### Шаг 5. Выгрузить всё в Excel

Жми `5`, формат `1` (Excel), Enter для папки.

В папке `export/` появятся файлы (открываются двойным кликом в Excel):

| Файл | Что внутри |
|---|---|
| `binance_spot_BTCUSDT_5m.xlsx` | Все 5-минутные свечи Binance Spot |
| `binance_futures_BTCUSDT_5m.xlsx` | То же для Binance Futures (с taker buy/sell) |
| `bybit_linear_BTCUSDT_5m.xlsx` | Bybit USDT-Perp |
| `analysis_binance_spot_outcomes.xlsx` | Каждая свеча с UP/DOWN исходом |
| `analysis_binance_spot_series.xlsx` | Все серии — старт/конец, длина, return |
| `analysis_binance_spot_signals.xlsx` | Все сигналы со всеми признаками |
| `analysis_binance_spot_long_series_context.xlsx` | Только длинные серии (≥7) с контекстом |
| `backtest_filters_report_binance_spot.xlsx` | **Главный отчёт** по бэктесту |
| `charts\01...07_*.png` | 7 диаграмм (см. ниже) |

### Что внутри `backtest_filters_report_*.xlsx`

Это главный файл — там 6 листов:

| Лист | Что показывает |
|---|---|
| **Выводы** | Авто-резюме на русском: база vs лучшая стратегия, walk-forward, сравнение ТЗ-vs-инверт, что делать дальше. **Открывай этот лист первым.** |
| **Сводка** | Полная таблица: каждая стратегия × 3 периода (весь / train / test). |
| **Описание_фильтров** | Что каждый фильтр делает, по-русски. |
| **Параметры** | Параметры стратегии (payout, ставки) и пояснения к колонкам Сводки. |
| **Sweep_по_payout** | Только если в шаге 4 ввёл список payout — таблица «стратегия × payout». |
| **Графики** | Все 7 PNG-диаграмм вставлены прямо в Excel. |

Все времена в отчётах — **UTC** (как на биржах), кроме колонок
`start_msk` / `end_msk` в `long_series_context.xlsx` и `hour_msk`
в `signals.xlsx` — там московское время.

## Графики (`export\charts\` + лист «Графики» в отчёте)

7 PNG-диаграмм, создаются автоматически при запуске бэктеста (`4`).
Каждый файл — обычный `.png`, можно открыть просмотрщиком картинок
или вставить в презентацию.

### 1. `01_stop_rate_by_hour_msk.png` — Доля проигрышей по часам МСК

Столбчатая диаграмма: ось X — часы суток (0–23 МСК), ось Y — доля
проигрышей (loss / total) в %. Горизонтальная пунктирная линия — среднее
по всем часам.

**Что искать:** часы со столбцом сильно выше среднего — в них концентрируются
длинные серии. Если 02:00–08:00 МСК (азиатская сессия) даёт заметный
прирост — это аргумент за фильтр «не входить в ночь».

### 2. `02_stop_rate_by_volume_bucket.png` — Доля проигрышей по группам объёма

Столбцы по группам `volume_ratio` (объём 3 свечей / 3× среднего за час
до триггера): `<0.5`, `0.5–0.8`, `0.8–1.0`, `1.0–1.2`, `1.2–1.5`,
`1.5–2.0`, `>2.0`. Над каждым столбцом — количество сигналов в группе.

**Что искать:** если левые столбцы (низкий объём) заметно выше правых —
это подтверждает «инверт»-гипотезу: длинные серии происходят в тихие фазы.
Если правые столбцы выше — подтверждается гипотеза ТЗ.

### 3. `03_stop_rate_by_taker_bucket.png` — Доля проигрышей по taker imbalance

То же, но по taker imbalance в направлении серии: `<0.45`, `0.45–0.50`,
…, `>0.65`. Над столбцами — количество сигналов.

**Что искать:** где сильнее доля проигрышей — при сбалансированном
тейкере (около 0.50) или при сильном перекосе (>0.60). Это прямой ответ
на тезис из ТЗ «большой объём в одну сторону в стакане → плохо».

### 4. `04_stop_rate_breakout_vs_no.png` — Пробой vs без пробоя

Две колонки: «Нет пробоя 60м» и «Есть пробой 60м». Над каждой — % проигрышей
и число сигналов.

**Что искать:** какая колонка выше. По нашему опыту — выше колонка
«Нет пробоя» (длинные серии чаще без импульса).

### 5. `05_pnl_by_3h_window_msk.png` — PnL по 3-часовым окнам МСК

PnL за весь период, сгруппированный по 8 окнам по 3 часа: 00–03, 03–06,
…, 21–24 (МСК). Зелёные столбцы — плюс, красные — минус. Над столбцами
PnL и количество сигналов.

**Что искать:** окна со стабильно красным PnL — кандидаты на фильтр
«не входить в это время».

### 6. `06_pnl_by_6h_window_msk.png` — PnL по 6-часовым окнам МСК

То же, но в 4 окна по 6 часов (азиатская / европейская / американская
утренняя / американская вечерняя сессия в первом приближении).

### 7. `07_long_series_length_distribution.png` — Распределение длин длинных серий

Сколько серий длиной ровно 7, ровно 8, ровно 9, …, ровно 15. Над каждым
столбцом — точное число.

**Что показывает:** насколько «толстый хвост» у распределения — много ли
у нас экстремальных серий (10+, 12+), или их редко. Если 90% длинных серий
— длиной 7–8, ситуация менее катастрофичная, чем кажется. Если есть много
12+ — стратегия особенно болезненна без фильтра.

## Передача результатов на анализ

### Способ 1 — Разработчику

В главном меню жми **`7`** — "Упаковать логи в ZIP для разработчика".

Создастся файл `B_parser_logs_YYYYMMDD_HHMMSS.zip` рядом с программой.
Внутри:
- Все логи сессий (что делал, какие ошибки случались)
- `cache_inventory.txt` — список файлов в кэше (без самих данных, ZIP лёгкий)
- `config.yaml` — какие параметры стояли
- `env.txt` — версия Python, ОС, путь к exe (для отладки)

Отправь этот ZIP разработчику в Telegram / email.

### Способ 2 — Через ChatGPT / Claude

Если хочешь обсудить результаты сам с LLM:

1. Прогони бэктест (шаги 1-4 выше), получи `backtest_filters_report_binance_spot.xlsx`
   и папку `export\charts\` с 7 PNG.
2. Открой ChatGPT (Plus) или Claude.ai.
3. **Загрузи xlsx прямо в чат** (значок скрепки → выбрать файл). Если
   LLM поддерживает загрузку картинок — приложи дополнительно 7 PNG
   из `export\charts\` (особенно полезны 01, 02 и 05 — по часам МСК,
   по объёму и PnL по окнам).
4. Скопируй этот промпт:

```
Это отчёт по бэктесту стратегии "мартингейл против серии" на Polymarket
BTC 5-минутных маркетах.

Стратегия: после 3 одинаковых исходов подряд (UP/DOWN) ставим против с
догоном 3 ставки (удвоение 1/2/4, payout 1.8). Тестировались данные
за 18.12.2025 — 07.05.2026 (около 5000 сигналов).

В xlsx 6 листов:
- "Выводы" — авто-резюме на русском (читай первым).
- "Сводка" — полная таблица: каждая стратегия × 3 периода (весь / train
  до 31.03 / test с 01.04).
- "Описание_фильтров" — что каждый фильтр делает.
- "Параметры" — payout, ставки, интерпретация колонок.
- "Sweep_по_payout" — таблица по разным значениям payout (если делал sweep).
- "Графики" — 7 диаграмм вставлены в Excel.

Фильтры делятся на:
- "ТЗ-1..5" — гипотеза "длинные серии = пробой + объём + taker imbalance"
- "Инверт-1..5" — обратная гипотеза "длинные серии = тихий рынок"
- "Час: ночь МСК" и "Комбо" — фильтр по времени суток

Помоги:
1. Выбрать стратегию с лучшим балансом train vs test (не переподогнанную).
2. Сравнить ТЗ-фильтры vs инверт-фильтры — какая гипотеза подтверждается.
3. По графикам "Stop rate по часам МСК" и "PnL по 3-часовым окнам" —
   стоит ли добавить более точный фильтр по времени суток?
4. По графику "Распределение длин длинных серий" — насколько толстый хвост?
5. По листу "Sweep_по_payout" — при каком payout стратегия становится
   стабильно прибыльной на test?
```

Дальше LLM сама прочитает отчёт и даст рекомендации.

## Если что-то не работает

- **HTTP 403 при пинге бирж** — включи VPN (Европа/США).
- **Скачивание aggTrades встаёт** — это нормально, файлы по 50-150 МБ
  на день. Программа автоматически возобновляет загрузку при обрывах.
  Можно перезапустить — кэш докачается с того же места.
- **Меню по-русски, но кракозябры** — переключи кодировку cmd:
  `chcp 65001`, потом запусти программу заново.
- **Программа закрылась с ошибкой** — нажми `7` (пакет логов) и пришли
  разработчику. В логах есть полная информация для разбора.

## Конфигурация (`config.yaml`)

По умолчанию:
```yaml
symbol: BTCUSDT
interval: "5m"
period:
  start: "2025-12-18T00:00:00Z"
  end:   "2026-05-07T23:59:59Z"
cache_dir: "cache"
```

Можно изменить:
- **symbol** — другая пара (например `ETHUSDT`)
- **period** — другой диапазон дат
- **cache_dir** — путь к кэшу (если хочешь на другой диск)

После редактирования сохрани файл в UTF-8 и перезапусти программу.
